# Student Counsellor Environment - Implementation Guide

## Overview
The Student Counsellor Environment is a complete OpenEnv implementation that provides a chatbot environment for evaluating the quality of counselling responses to students with low confidence, exam stress, and self-doubt.

## Files Created/Modified

### 1. **tasks.py** - Task Definitions
Defines three counselling tasks of varying difficulty:

- **Easy**: "I am bad at studies"
  - For students with basic self-doubt about academic abilities
  - Expected: Empathy + encouragement + practical study tips

- **Medium**: "I feel I will fail exams"
  - For students with exam anxiety and fear of failure
  - Expected: Deep empathy + strong encouragement + structured advice

- **Hard**: "Everyone is better than me"
  - For students experiencing severe peer comparison and inadequacy feelings
  - Expected: Strong empathy + challenge comparison mindset + practical growth advice

```python
Task = dataclass with:
- task_id: str
- input_message: str
- description: str
- difficulty: Literal["easy", "medium", "hard"]
- expected_behavior: str
```

### 2. **grader.py** - Reward Calculation Engine
Implements deterministic grading (0.0 to 1.0) based on four criteria:

#### Empathy Keywords (35% weight)
- "i understand", "it's okay", "i'm sorry", "that sounds difficult", etc.

#### Encouragement Keywords (35% weight)
- "you can do it", "you are capable", "keep trying", "don't give up", etc.

#### Practical Advice Keywords (30% weight)
- "make a plan", "start small", "practice daily", "revise", "take breaks", etc.

#### Safety Checks
- Bad keywords = IMMEDIATE REWARD 0: "you will fail", "you are useless", "no hope", etc.
- Context-aware: "don't give up" is positive (not counted as bad)

**Grading Formula:**
```
empathy_score = min(1.0, empathy_count * 0.3) if empathy_count > 0 else 0.0
encouragement_score = min(1.0, encouragement_count * 0.3) if encouragement_count > 0 else 0.0
advice_score = min(1.0, advice_count * 0.25) if advice_count > 0 else 0.0

total_reward = (empathy_score × 0.35) + (encouragement_score × 0.35) + (advice_score × 0.30)
total_reward = clamp(total_reward, 0.0, 1.0)
```

### 3. **env.py** - Core Environment Implementation
Implements the OpenEnv Environment interface with three main methods:

#### reset() → StudentCounsellorObservation
- Resets the environment
- Selects a task (currently cycles through all)
- Returns task details and student message

#### step(action: StudentCounsellorAction) → StudentCounsellorObservation
- Evaluates the counsellor's response using grader.grade()
- Marks episode as done after one step
- Returns reward and response evaluation

#### state → State
- Returns current episode_id and step_count
- Maintains OpenEnv compatibility

**Observation Fields:**
- student_message: The student's concern
- task_id, task_difficulty, task_description: Task metadata
- expected_behavior: What a good response should include
- counsellor_response: The response being evaluated
- reward: Grade from grader (0.0-1.0)
- metadata: Additional details including reward_details dict

## Key Design Decisions

1. **Deterministic Task Selection**: Currently cycles through "easy" task for consistency. Can be modified to support task_difficulty parameter in reset().

2. **One-Step Episodes**: Each episode has exactly one step (student message → response evaluation). This aligns with the grading model.

3. **Context-Aware Grading**: Bad keyword detection uses regex to avoid false positives (e.g., "don't give up" is positive).

4. **Weighted Scoring**: Empathy and encouragement are equally important (35% each), practical advice is 30%.

5. **Keyword-Based Evaluation**: Simple, deterministic, and interpretable. Can be extended with more sophisticated NLP.

## Extension Points

1. **Task Difficulty**: Modify env.py reset() to accept difficulty parameter
2. **Grader Enhancement**: Add more keywords or implement semantic similarity
3. **Multi-turn Conversations**: Extend to multiple steps per episode
4. **Logging**: Add detailed reward breakdown logging
5. **Custom Models**: Easy to plug in different LLM backends in inference.py

## Error Handling

- Missing OPENAI_API_KEY: Clean error message suggesting setup
- Episode already done: ValueError with helpful message
- No task selected: RuntimeError with reset() suggestion
- OpenEnv not installed: Graceful degradation in __init__.py
